# 开源社区核心开发者台账系统

**Authors:**

- @ZhengZhenyu
- @zeshengzong

## 1 简介

设计开发开源社区核心开发者台账系统，为社区运营人员提供Ascend相关社区核心开发者的行为数据与趋势，为Ascend软件分发推送、用户体验感知、新项目和潜在合作机会的发现等提供参考和输入。

## 2 动机

- AI领域开源项目数量庞大，各自独立发展但又有广泛性联系，只看一个社区无法识别其中联系；
- Ascend领域生态发展过程中，核心用户和开发者是重要发展要素，通过系统性收集和分析开发者行为数据，有利于识别和更有针对性的围绕开发者开展运营工作。同时关注核心开发者的动向可为参与开源项目提供输入指导；
- 在内容传播上，不同项目的受众和渠道有所区分，需要收集有效数据分析用户、开发者，确保Ascend技术内容可以快速传递到对应用户手中

## 3 需求细节

### Use Case 1: 查询指定社区的开发者贡献情况列表

社区运营人员通过UI前端和接口查询指定社区的开发者社区参与情况，查询返回内容的每一条记录包括`GitHub_ID`, `GitCode_ID(若有)`, `姓名`, `组织`, `技术贡献`, `活动贡献`, `内容贡献`。

### Use Case 2: 查询鲲鹏或Ascend领域开发者的整体情况列表

社区运营人员通过UI前端和接口查看鲲鹏或Ascend社区开发者的整体情况，包括整体D0、D1、D2开发者人数。

### Use Case 3: 查询指定开发者的贡献情况详情

社区运营人员通过UI前端和接口查询指定的开发者代码托管平台ID或姓名，参与的详细情况，查询返回内容的每一条记录包括`GitHub_ID`, `GitCode_ID(若有)`, `姓名`, `组织`, `参与社区`。

### Use Case 4: 查询指定社区所开展活动的列表

社区运营人员通过UI前端和接口查询指定社区的名称查询该社区的活动列表，查询返回内容的每一条记录包括`活动名称`, `活动时间`, `活动地点`, `参与人数`。

### Use Case 5: 查询指定社区活动的执行细节情况

社区运营人员通过UI前端和接口指定社区活动名称查询该次社区活动详情，查询返回内容的包括`活动名称`, `活动时间`, `活动地点`, `参与人数`，`议题及讲师`，`与会核心开发者`，`商业线索`，`反馈问题`，`问题闭环情况`。

### Use Case 6：录入社区活动及社区活动执行细节

社区运营人员通过UI前端和接口创建活动记录，并录入社区活动的`活动名称`, `活动时间`, `活动地点`, `参与人数`，`议题及讲师`，`与会核心开发者`，`商业线索`，`反馈问题`，`问题闭环情况`详细信息，录入后信息可进行后续编辑。
